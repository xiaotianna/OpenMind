# AI Agent Desktop - Codex 协作规范

本文件用于指导 Codex 在本仓库中的行为。目标是与 `CLAUDE.md` 保持一致，并补充可执行约束。

## 1. 项目概述

- 项目名称：AI Agent Desktop
- 技术栈：Electron + React + TypeScript（Monorepo）
- 目标：构建 AI Agent 桌面客户端，支持对话、任务处理与本地能力集成

## 2. 基本执行原则

- 先读后改：先理解上下文，再做最小必要修改
- 优先复用：优先复用现有组件、工具函数和样式约定，避免重复造轮子
- 改后验证：改动后运行必要的检查（类型、lint、测试）并反馈结果
- 明确风险：如有未验证路径或潜在回归，需在结果中明确标注

## 3. 禁止事项

- 禁止自动启动开发服务器（如 `pnpm run dev`），除非用户明确要求
- 禁止私自执行 `git commit`、`git push`，除非用户明确要求
- 禁止做无关重构或大范围格式化
- 禁止修改与当前任务无关的文件

## 4. 代码组织与拆分

### 4.1 组件拆分

- 不为“可能复用”而提前抽象
- 当相同或高度相似逻辑出现两次及以上，再考虑抽离
- 保持单个组件职责清晰，避免过深嵌套

### 4.2 通用逻辑拆分

- 可复用工具函数放 `src/lib` 或 `src/utils`
- 可复用状态与副作用逻辑放 `src/hooks`
- 命名语义化，避免 `handleData`、`tempFn` 之类模糊命名

## 5. 文件与命名规范

- 目录名：`kebab-case`
- React 组件文件：`PascalCase.tsx`
- Hooks 文件：`use-*.ts`（或项目现有风格下的 camelCase）
- 工具函数：camelCase
- 样式文件：`kebab-case`

### 5.1 注释规范

- 代码注释统一使用中文
- 注释末尾不加句号

## 6. 路由与页面组织

- 路由集中在 `src/router/index.tsx` 管理
- 页面放在 `src/pages`
- 同类页面按目录聚合，父布局优先使用 `index.tsx`

## 7. UI 与交互规范

- 保持与当前项目视觉风格一致（间距、圆角、层级、配色）
- 分组列表使用清晰分组标题与统一间距节奏
- 动画优先使用 Framer Motion，时长建议 `0.2s - 0.3s`
- 动效克制，服务于可用性，不堆叠炫技动画

## 8. 质量门槛

- TypeScript 不引入新的类型错误
- 尽量不引入 `any`；必须使用时说明理由与边界
- 异步流程要有错误处理和兜底提示
- 涉及用户输入、文件路径、外部响应时做基本校验

## 9. 提交规范（仅在用户要求提交时）

- 提交信息使用中文
- 遵循 Conventional Commits：`feat|fix|refactor|docs|style|chore`
- 示例：`feat: 增加会话列表搜索`
- 单行不超过 100 字符

## 10. Codex 输出约定

- 输出先给结果，再给关键变更点
- 引用文件时提供明确路径与必要行号
- 若未执行某项验证，需明确说明“未执行”及原因

## 11. 优先级规则

当多份规范冲突时，按以下优先级执行：

1. 用户当前指令
2. 仓库根目录 `AGENTS.md`
3. `CLAUDE.md` 中与实现相关的通用工程规范
